## Sprint 3

### Sprint Goal
Integrate the **Frontend**, **Backend**, and **LLM API** into a unified working system.  
Develop a new and improved frontend, introduce **compression** and **dialect changes** in the backend, and ensure seamless interaction between all components.

---

### Order of Completion

#### Week 1
1. **Feedback Incorporation**
   - Implemented UI/UX improvements based on Sprint 2 feedback.
   - Fixed minor bugs and layout inconsistencies.
2. **Create FastAPI Endpoint**
   - Built FastAPI endpoints for story generation and user interactions.
   - Integrated endpoints with LLM API for dynamic storytelling.
3. **Improve Initial Frontend**
   - Enhanced layout and visual structure for better usability.
   - Started development of a new theme and refined component design.
4. **Complete Story-Related Backend**
   - Finalized backend logic for story creation, storage, and compression.
   - Added dialect and tone control features for story generation.

---

#### Week 2–3
1. **Completed New Frontend**
   - Developed and finalized new user interface and theme.
   - Improved navigation, responsiveness, and storytelling experience.
2. **Completed Basic Storyteller**
   - Implemented functional storytelling system integrated with LLM API.
   - Enabled prompt customization and dynamic story generation.
3. **Completed Backend and API Integration**
   - Successfully integrated FastAPI backend with frontend modules.
   - Established seamless data flow between UI, backend, and LLM API.
