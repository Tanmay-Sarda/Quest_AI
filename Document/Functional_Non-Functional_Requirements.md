# Functional Requirements

1. **Register**  
**Reason:** To login users should register first.  
**Elicitation Technique:** Prototyping  
**Reason for elicitation technique:** This one of the standard features for this app. Prototyping because while creating the basic paper prototyping for the app we get to know this must be included in the app flow.  

2. **Login**  
**Reason:** To save the progress of the story and continue where left.  
**Elicitation Technique:** Prototyping  
**Reason for elicitation technique:** This is one of the standard features for this app. Prototyping because while creating the basic paper prototyping for the app we get to know this must be included in the app flow.  

3. **Create new story**  
**Reason:** To create a new story from scratch.  
**Elicitation Technique:** Brainstorming  
**Reason for elicitation technique:** This is the basic and obvious feature for our application.  

4. **Logout**  
**Reason:** To login users should register first.  
**Elicitation Technique:** Prototyping  
**Reason for elicitation technique:** This one of the standard features for this app. Prototyping because while creating the basic paper prototyping for the app we get to know this must be included in the app flow.  

5. **Story form**  
**Reason:** To choose or create character and settings.  
**Elicitation Technique:** Prototyping, Brainstorming  
**Reason for elicitation technique:** Prototyping is the most effective technique here because it's a visual and interactive way to show the user what they will be interacting with. A brainstorming elicitation technique is being applied because we got this requirement during brainstorming. "Create new story" requirement needs creative and collaborative ideas to define what character the user will use and at what setting the user wants to continue.  

6. **Story generation page**  
**Reason:** Here the user will enter the prompt or the flow of the story.  
**Elicitation Technique:** Prototyping, Brainstorming  
**Reason for elicitation technique:** This functionality is similar to the last one story form. So, the reason for using this elicitation technique in this functional requirement is the same as the last one.  

7. **View previous story**  
**Reason:** Here users will be able to access the previously created stories and will also be able to continue those stories.  
**Elicitation Technique:** Brainstorming  
**Reason for elicitation technique:** During brainstorming we got this idea that user must be able to access their previous stories.  

8. **Exit (This exit is for exiting the current story)**  
**Reason:** To login users should register first.  
**Elicitation Technique:** Prototyping  
**Reason for elicitation technique:** This one of the standard features for this app. Prototyping because while creating the basic paper prototyping for the app we get to know this must be included in the app flow.  

9. **Public stories**  
**Reason:** To view stories created by others and also share self created stories.  
**Elicitation Technique:** Brainstorming, Survey  
**Reason for elicitation technique:** Brainstorming sessions with stakeholders and potential users help generate creative ideas on how stories should be displayed, discovered, and interacted. Since this feature is audience-centric, a large user base needs to be consulted. Surveys allow gathering quantitative data on user preferences.  

10. **Multiplayer**  
**Reason:** More than one user can play a different role in one story and create a story.  
**Elicitation Technique:** Brainstorming, Survey  
**Reason for elicitation technique:** This idea came in one of the brainstorming sessions of our group. We also took the survey for the feedback on the features of our application and the audience liked this feature the most.
